1.1

Class Basics

A class defines what an object looks like and what it can do. Every object created from a class gets its own copy of the data.

Data & Behaviour

Every class has two parts

Data Members
Variables that hold the object's data
Also called: attributes, fields
Member Functions
Functions that define what the object can do
Also called: methods, behaviours
Employee class — full example from your course
class Employee {
    // Data Members — private
  private:
    int    id;
    float  salary;
    string name;

    // Member Functions — public
  public:
    Employee() {           // Default Constructor
        id     = 0;
        salary = 0;
        name   = "";
    }

    Employee(int i, float s, string n) {  // Parameterized Constructor
        id     = i;
        salary = s;
        name   = n;
    }

    void printInfo() {
        cout << "ID = "     << id     << endl;
        cout << "Salary = " << salary << endl;
        cout << "Name = "   << name   << endl;
    }
};

int main() {
    Employee employee1;                         // uses default constructor
    Employee employee2(1, 1000, "Ahmed");      // uses parameterized constructor
    employee2.printInfo();
    return 0;
}
1.2

Access Specifiers

Two keywords that control who can see and use each member of the class.

public & private

The two specifiers you use right now

SpecifierWho can access it?Used for
publicAnywhere in the programMember functions, constructors
privateOnly inside the class itselfData members
💡
The rule you always follow: data members → private, member functions → public. This protects the data from being changed directly from outside the class.
What happens when you try to access private data
Employee emp(1, 1000, "Ahmed");

emp.salary = 5000;    // ✘ ERROR — salary is private
emp.printInfo();      // ✔ OK — printInfo() is public
1.3

Constructors & Overloading

A constructor runs automatically when an object is created. It sets the starting values for the data members.

?
Why This Matters
What happens if you create an object and never set its data?
The data members will hold random leftover values from memory — not zero, not empty. If you then call printInfo(), the output will be meaningless. A constructor makes sure every object starts with known values right away.
Constructor Rules

What makes a constructor different

1
Same name as the class
If the class is Employee, the constructor is also Employee
2
No return type — not even void
This is how C++ knows it is a constructor, not a regular function
3
Called automatically
Runs the moment you write Employee emp; — you never call it yourself
4
Always public
So that code in main() can create objects
Constructor Overloading

Default vs Parameterized

You can have more than one constructor in the same class, as long as they have different parameters. This is called constructor overloading.

Default Constructor
No parameters — sets everything to a starting value
Called when you write: Employee emp;
Parameterized Constructor
Takes values and assigns them directly
Called when you write: Employee emp(1, 1000, "Ahmed");
Both constructors side by side
// Default — no arguments
Employee() {
    id     = 0;
    salary = 0;
    name   = "";
}

// Parameterized — receives values
Employee(int i, float s, string n) {
    id     = i;
    salary = s;
    name   = n;
}

// In main():
Employee emp1;                         // → default  : id=0, salary=0, name=""
Employee emp2(1, 1000, "Ahmed");      // → parameterized : id=1, salary=1000
⚠️
Always assign values in the constructor. If you skip this, the object's data contains random values. Printing it or using it before setting values is a common source of errors.
📌
Watch your output format. When printing, endl moves to a new line and flushes the output. "\n" also moves to a new line but does not flush. Use whichever the question asks for — if not specified, endl is the safer choice.
1.4

Destructor

The destructor is the opposite of the constructor — it runs automatically when an object is destroyed.

How It Works

Automatic cleanup when an object's life ends

When an object goes out of scope — for example, when a function ends — C++ automatically calls the destructor. You don't call it yourself.

  • Same name as the class, with ~ before it — e.g. ~Employee()
  • No parameters and no return type
  • Only one destructor per class — it cannot be overloaded
  • For most classes in this course, you don't need to write one — C++ handles it
Constructor and destructor — lifecycle
class Employee {
  public:
    Employee()  { cout << "Employee created"  << endl; }
   ~Employee() { cout << "Employee destroyed" << endl; }
};

int main() {
    Employee emp;     // → "Employee created"
    return 0;
}                      // → "Employee destroyed"  (main ends, emp is gone)
2.1

Struct Basics

A struct groups related variables of different types together under one name. Each variable you create from it gets its own independent copy of the data.

Defining & Using

The Employee as a struct

The key strength of a struct: members can be different data types — int, float, string, all in the same struct.

Two ways to assign values
struct Employee {
    int    id;
    string name;
    float  salary;
};

// Way 1 — dot operator
Employee emp1;
emp1.id     = 101;
emp1.name   = "Sara";
emp1.salary = 8000;

// Way 2 — curly braces {}
Employee emp2 = {102, "Ali", 9000};
💡
When using {}, the values go in the same order as the members are declared in the struct. id first, then name, then salary.
2.2

Arrays & Nesting

Structs support two useful features: storing an array as a member, and putting one struct inside another.

Array Inside Struct

Store multiple values of the same kind in one member

For example, a Student has marks for 3 subjects — instead of 3 separate variables, use one array member.

Array as a struct member
struct Student {
    string name;
    float  marks[3];    // array member — 3 subject marks
};

Student s;
s.name      = "Ali";
s.marks[0]  = 85;     // first subject
s.marks[1]  = 90;     // second subject
s.marks[2]  = 78;     // third subject
Struct Inside Struct

Use another struct as a member type

For example, a Student has an Address — and Address itself has multiple fields.

Nested structs
struct Address {
    string city;
    string street;
};

struct Student {
    string  name;
    Address address;   // Address struct is a member
};

Student s;
s.name           = "Sara";
s.address.city   = "Cairo";   // two dots for nested access
s.address.street = "Nile St";
💡
To reach a member inside a nested struct, you use two dot operators — one to enter the outer struct, one to enter the inner struct: s.address.city
2.3

Struct Operations

Two things you can do with struct variables — and one thing you absolutely cannot.

Assignment — Works Fine

Copying one struct into another

You can assign one struct variable to another of the same type using =. This copies every member at once — no need to copy them one by one.

Assignment
Employee emp1 = {101, "Sara", 8000};
Employee emp2;

emp2 = emp1;    // copies all fields — emp2 is now an independent copy

emp2.name = "Ali";    // changing emp2 does NOT affect emp1
cout << emp1.name;    // → Sara  (unchanged)
Comparison — Read Carefully

You cannot use == on structs

C++ does not support comparing two struct variables directly with ==. You must compare each member one by one.

Right vs wrong
// ✘ DOES NOT WORK — compile error
if (emp1 == emp2) { ... }

// ✔ CORRECT — compare member by member
if (emp1.id     == emp2.id     &&
    emp1.name   == emp2.name   &&
    emp1.salary == emp2.salary) {
    cout << "Same employee";
}
⚠️
Common exam trap. You can copy a struct with = but you cannot compare two structs with ==. Always compare member by member.

Function Basics

A function is a named block of code that does one job. You write it once and can use it as many times as you need.

Structure of a Function

The four parts

Anatomy of a function
//  return type   name      parameters
       int         add   (int a, int b)
{
    return a + b;    // result sent back to caller
}
Return Type
The type of value the function sends back
Use void if the function returns nothing
Function Name
How you call the function from other places
Should describe what the function does
3.2

Global vs Local Variables

Where a variable is declared decides where it can be used.

Scope

Two types — different visibility

TypeWhere declaredWhere visible
GlobalOutside all functionsAnywhere in the program
LocalInside a function or blockOnly inside that function
Global vs local — what each can see
int total = 0;              // global — visible everywhere

void addScore(int score) {
    int bonus = 10;          // local — only visible inside addScore()
    total += score + bonus;
}

int main() {
    addScore(50);
    cout << total;            // → 60  (global, accessible here)
    cout << bonus;            // ✘ ERROR — bonus is local to addScore()
}
⚠️
A local variable only exists while that function is running. Once the function ends, the variable is gone. You cannot use it anywhere outside that function.
3.3

Parameters vs Arguments

Two words that look similar but mean different things — one is in the definition, one is in the call.

The Difference

Easy to mix up — here is the clear distinction

Parameter
The variable in the function definition
It is a placeholder — it receives a value when the function is called
Argument
The actual value passed when calling the function
It is the real data — what gets copied into the parameter
Parameters vs arguments — labeled
//         parameters ↓ ↓
void printSalary(string name, float salary) {
    cout << name << ": " << salary << endl;
}

int main() {
//              arguments ↓         ↓
    printSalary("Ahmed",      5000);
}
💡
Simple rule: if you see it in the function header between ( ) with a type — it's a parameter. If you see it in the function call — it's an argument.
3.4

Pass by Value & Pass by Reference

When you pass a variable to a function, C++ can either send a copy or send the original.

Pass by Value

The function gets a copy — the original is safe

This is the default in C++. The function works on its own copy of the value. Changes inside the function do not affect the original variable.

Pass by value
void addBonus(float salary) {   // salary here is a copy
    salary += 500;
    cout << salary;              // → 8500
}

int main() {
    float mySalary = 8000;
    addBonus(mySalary);
    cout << mySalary;            // → 8000  (unchanged)
}
Pass by Reference

The function works on the original — changes stick

Add & before the parameter name. Now the function receives the actual variable — not a copy. Any change inside the function changes the original.

Pass by reference — note the &
void addBonus(float &salary) {  // & means: work on the original
    salary += 500;
}

int main() {
    float mySalary = 8000;
    addBonus(mySalary);
    cout << mySalary;            // → 8500  (changed!)
}
Pass by Value
Function gets a copy
Original variable stays unchanged
Pass by Reference &
Function works on the original
Changes made inside the function stay after it ends
✅
The only difference in the code is the & symbol in the parameter. float salary = by value. float &salary = by reference. That one symbol changes everything.
